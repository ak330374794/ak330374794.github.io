深入浅出 Tailwind CSS 性能优化技巧

Alex Chen
数字游民 / 前端工程师

Tailwind CSS 已经成为现代 Web 开发的首选工具之一。它通过原子类（Utility-first）的方式极大地提高了开发效率，但如果不加节制地使用，也可能带来性能隐患。本文将探讨如何在保持开发体验的同时，优化最终产物的性能。

1. 启用 PurgeCSS (Just-in-Time 模式)

在 Tailwind v3 之前，我们需要手动配置 purge 选项。而现在，JIT 引擎是默认启用的。它会扫描你的 HTML、JavaScript 模板，并仅生成你实际使用到的 CSS 类。

“JIT 模式的引入是 Tailwind 历史上最大的飞跃，它让生成的 CSS 体积从几百 KB 骤降到几 KB。”

2. 避免动态拼接类名

这是新手最容易犯的错误。Tailwind 的扫描器是基于静态分析的，如果你在代码中这样写： 

// ❌ 错误做法
const color = 'red';
return <div className={`text-${color}-500`}>Hello</div>;

// ✅ 正确做法
const colorClasses = {
  red: 'text-red-500',
  blue: 'text-blue-500'
};
return <div className={colorClasses[color]}>Hello</div>;

3. 使用组件提取常用模式

虽然 Tailwind 鼓励原子类，但对于高度重复的 UI 模式（如按钮、输入框），你应该使用模板组件（React/Vue 组件）或者 @apply 指令。

通过合理的架构设计，我们可以确保项目在规模增长时，CSS 的体积依然保持线性甚至零增长。这正是 Tailwind 最迷人的地方。
